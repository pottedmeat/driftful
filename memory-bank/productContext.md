# Product Context

## Why Driftful Exists

Driftful was born from recognizing two key user needs:
1. The desire for a **simpler digital task manager** that doesn't overwhelm users with features
2. The need to **bridge the gap between analog and digital journaling** for Bullet Journal enthusiasts

## Core Problems Solved

### 1. Feature Overwhelm
Traditional task managers like Trello or Notion often provide too many features, creating cognitive overhead. Driftful intentionally limits features to maintain:
- Quick entry without complex forms
- Simple organization without nested hierarchies
- Clear navigation without deep menus
- Focused interface without distracting options

### 2. Lost Tasks and Intentions
Users often struggle with:
- Tasks getting buried in long lists
- Important items being forgotten
- Difficulty in tracking task history
- No clear way to rediscover old items

Driftful solves these through:
- Automatic migration of incomplete tasks
- Clear breadcrumb trails for task movement
- Page-based organization for context
- Simple navigation between timeframes

### 3. Digital vs. Analog Balance
Many users love physical Bullet Journals but need digital convenience. Driftful bridges this gap by:
- Maintaining the simplicity of pen and paper
- Following BuJo principles like migration
- Preserving the "permanent ink" metaphor
- Supporting rapid capture like a notebook

## How It Should Work

### 1. Entry Flow
- **Mobile:** Persistent bottom input bar
  - Always visible for quick entry
  - One-tap access to parent items
  - Immediate item creation
- **Desktop:** Floating action button
  - Quick click for new items
  - Long press for migration options
  - Modal-based entry

### 2. Organization Model
- **Pages:** Group items by date range
  - One "current" page
  - Multiple past pages
  - User-controlled page creation
- **Timeframes:** Broader context views
  - Week view for short-term planning
  - Month view for medium-term overview
  - Year view for long-term tracking
- **Collections:** Non-temporal organization
  - Independent from timeframes
  - Simple list structure
  - No nested hierarchies

### 3. Migration Philosophy
- **Automatic Migrations:**
  - Occur during page creation
  - Based on item age
  - No confirmation needed
  - Create breadcrumb trails
- **Manual Migrations:**
  - Simple one-tap process
  - No confirmation dialogs
  - Clear breadcrumb tracking
  - Maintain item history

### 4. State Management
- **Task States:**
  - Complete/Incomplete toggle
  - Star for significance
  - Strikethrough for irrelevance
- **Past Items:**
  - Read-only in past pages
  - Require copying to modify
  - Maintain original history
  - New copies start fresh

## User Experience Goals

### 1. Speed and Simplicity
- Rapid item entry
- Minimal interface friction
- Clear, immediate feedback
- No unnecessary confirmations

### 2. Clarity and Context
- Clear item organization
- Easy timeframe navigation
- Visible item history
- Intuitive migrations

### 3. Reliability and Trust
- Offline-first operation
- Automatic synchronization
- No data loss
- Consistent behavior

### 4. Mindful Organization
- User-driven page creation
- Intentional migrations
- Clear history tracking
- No automated cleanup

## Success Indicators

### User Behavior
- Quick task entry (< 3 seconds)
- Regular page creation
- Active use of migrations
- Minimal task loss

### User Sentiment
- Feels simple and uncluttered
- Trust in item organization
- Comfort with migration system
- Satisfaction with minimal features

This context guides all product decisions, ensuring Driftful remains true to its mission of providing a simple, effective digital journaling experience that respects both minimalist principles and Bullet Journal methodology.
